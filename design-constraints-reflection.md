What are common technical constraints that impact UX design? 

Technical constraints are one of the biggest constraints that a UX designer has to deal with in their profession. Technical constraints are fairly ubiquitous and a competennt UX designer should be able to work successfully work around these constraints, instead of being hindered. 
One of the most common technical constraints is development feasibility and level of effort needed to implement a design. A UX designer might design an experience that provides users with great value, but if it requires the development team 6-9 months of work, then the cost cannot be justified with the type of value that is returned. 
Similarly, there are also issues with technical architecture. For example, a new collaborative real time body doubling feature might be recommended by a designer for focus bear. However, if the system cannot support real time collaborations without having significant changes to the back end, then this might not necessarily be feasible.  
On the other hand, limitations with a particular platform might affect how design decisions are made. For example, a feature for focus bear that has good results on desktop might be difficult to execute on mobile due to lack of screen real estate, issues with operating systems on mobile and other issues. 
Similarly, performance constraints need to be considered by UX designers as well as they make their design decisions. An interface that is highly interactive and that has complex interactions might be visually appealing, but this could result in slowing an application down or also create accessibility issues on devices that are not as powerful. 
This is particularly the case for low end devices. There are also time and budget constraints. A design decision might be appropriate from a UX designers perspective, but if the company is needing to release a simpler version first for a phase 1 launch and then make further iterations in phase 2 and phase 3, then needs to be accounted for as well. 
An example of this could be implementation of an onboarding experience that is adaptive in nature to each individual user in real time. 
This might provide users with a user experience that can be categorised as “exemplary”, but if developers feel that it requires a significant amount of backend work and is simply not feasible with the established they have in place, it won’t be actioned. In this instance, the team might suggest drastically simplifying the user interface.


How do UX designers collaborate with developers to find workable solutions? 

Collaboration is an integral part of being a UX designer. UX designers often collaborate with developers. This usually takes the form of discussing user needs and technical limitations that can impact design decisions made by the UX designer. 
The developer tries to convey to the UX designer what limitations might exist in implementing a certain design. When this occurs, the designer needs to understand that they need to work with the developer to find a middle ground solution in the face of various constraints. 
For example, a UX designer might suggest a specific interaction pattern for users because it significantly improves usability for users by reducing friction.  
However, after going through the designs, the developer might conclude that implementing the designs will need a significant level of effort or is simply can’t be achieved due to various other constraints. 
When this occurs, a competent UX designer would work with the developer to minimise complexity in order to implement the design after modifying it to a realistic level that can be implemented by the developer. 
This would mean that the designer would have to iterate on their designs again, conduct user testing and then report back to the developer to discuss how they implemented the feedback provided by the developer, whilst also ensuring that user goals and needs are not compromised. 



What are best practices for simplifying complex designs without losing functionality? 

Designs may need to simplified at times due to various constraints. The goal is to make things simpler, without compromising functionality. 
A common way to do this is by focusing on a users primary task and then removing anything that does not entirely help them out to complete the task. 
For example, if a user is attempting to create their focus bear routine, they should not be distracted by anything else while they attempt to accomplish this task. 
Another way to simplify complex designs is by breaking down a complex user flow into steps that are more manageable. 
It becomes much easier for a user to take action when things are broken down into steps for for them instead of being presented with too many options on a single screen. 
Designers need to also prioritise features based on how often those features are used. May feature that is used frequently needs to be accessed with more ease by the user. 
Any feature that is not used  as frequently can be accessed via being in secondary screens or settings screen. 
A lot of the time, many products can become quite complex because of the number of features offered without failing to consider how much value it provides to the users. 
Within the context of focus bear app, it might be useful to provide users with more streamlined configuration options instead of overwhelming them with too many. 
Offering users a default configuration state is also helpful to make things more functional and easier to use. 
The users can then be told via instructions that are Succinct and intuitive that the settings can be customised.



How can designers advocate for usability and accessibility, even when technical limitations exist? 

A designer can advocate for usability and acessibility by focusing on an underlying user need instead of trying force things by focusing on a particular type of design solution. 
Technical limitations are always going to exist alongside many other types of other limitations. It is important that UX designers focus on user needs when designing. 
User testing might be showing that users are struggling greatly to complete the overall onboarding process. 
If a designer comes up with a solution that is not technically feasible, such  as implementing motion graphics, the designer can still advocate for the solution by explaining how adopting the new recommendation will impact the completion rate for the onboarding process. 
Using evidence from analytics, user testing sessions, user interviews and accessibility guidelines will also help the UX designer advocate for their recommendations.  
Competent UX designers work around limitations by focusing on a way to come up with a solution that is workable rather than aiming for the ideal. 



If a developer tells you a design is too complex to implement, how would you respond? 

If a developer tells me that my designs are too complex to implement. I would firstly seek to gain an understanding of why this is the case. 
It is always important to understand more about the constraints that exist. This is important because I need to pinpoint if the issue exists because of limitations with the operating system, due to the level of effort it takes to develop the designs, device performance issues or other constraints related to timeframe and budget. 
Once I have properly pinpointed the issue, I can then explain to the developer the user problem that I am trying to solve.i can explore solutions that are more suitable that help solve the user problem whilst also ensuring that technical complexity is minimised. 
If I have designed a highly flashy and interactive onboarding for focus bear and I have been told by the developer that it is too complex. I can then simplify things by reducing the interactions and making things more simplified. 
It is Important to work collaboratively with the developers instead of defending my design decisions. This will not help the product goals or the user goals.



What are some non-visual factors (e.g., performance, accessibility, load time) that can impact a UX design? 

There are many non visual factors that can have an impact on UX design. A product can look very visually appealing and still have sub optimal UX, and be quite difficult to use. One of the biggest examples of this performance. 
If the screens take a long time to load, if everything feels slow, the buttons are not as responsive and fail to provide immediate user feedback after a user interacts with it, or if the app closes when a user interacts with it, then this can take away from the visual appeal of the product. 
Another issue is accessibility. Many users struggle if a product is hard to navigate using a keyboard, is incompatible with screen readers, or fails to adequately accommodate users with disabilities. 
Furthermore, if a product is glitchy and crashes when a user tries to use it, this can lead to user frustration and loss of trust with the product. 


How can UX designers help find middle-ground solutions between ideal design and technical feasibility? 

UX designers can find middle ground solutions by focusing on the user problem instead of trying to focus on a specific design solution or direction. For example, a UX designer might advocate for a real time body double feature for the focus bear app that helps people collaborate in real time. 
However, a developer might indicate that this is not possible to implement due to various issues with budget, time and level of effort needed, and technical constraints. 
The UX designer in such an instance can work with the developer in order to come up with simpler alternatives that help users with body doubling using the focus bear app. 
It’s always important to understand how technical constraints can impact what particular parts of user experience are important to users and come up with alternative ways to improve the user experience by working around technical constraints.
